## Review ownership

| Area | Review focus |
|---|---|
| Corpus / patterns | Synthetic scope, invariant, vulnerable/protected behavior, trace evidence. |
| Target adapters | Authorization model, no network by default, metadata, artifact redaction. |
| Reports / validation | Machine-readable evidence, schema stability, reproducibility. |
| Documentation | Current-vs-planned accuracy, no overclaiming, responsible-use framing. |
| Release / CI | Version consistency, examples validation, package build, security posture. |

## Response expectations

This is an early-stage open-source project. Issue and PR response times are best effort.
Security reports should use the private vulnerability reporting path in [SECURITY.md](SECURITY.md).
